Annotations
apiDog provides the following additional annotations to the those existing in apiDoc.
@apiChapter
Usage:
@apiChapter chapter_name

Defines to which chapter the doc block belongs.

The chapter will be shown as a navigation switcher (@html template). If @apiDefine declares definition with the same name also includes its title and description.

Example:
/** * @apiChapter Some chapter */

Example of usage with @apiDefine:
/** * @apiDefine chapter Some chapter * @apiDescription Some description */ /** * @apiChapter chapter */

@apiContentType
Usage:
@apiContentType content_type

Defines content type.

Used as a filter of the @apiExample content having corresponding content_type. Defaines the format for sending data by the "Send sample request" plug-in.

Can be defined multiply.

Example of application/x-www-form-urlencoded:
/** * @apiContentType form */

Example of application/json:
/** * @apiContentType json */

Example of application/xml:
/** * @apiContentType xml */

@apiErrorValue
Usage:
@apiErrorValue [{type}] value [description]

Describes custom error value.

Example:
/** * @apiErrorValue {String} value String value */

@apiFamily
Usage:
@apiFamily identifier_name

Defines unique identifier identifier_name of the doc block within its chapter, group and subgroup.

Used to distinguish between several doc blocks with the same descriptors to show them separately or combine the different doc blocks under versioning.

Example 1:
/** * @api {post} /test * @apiVersion v1 * @apiFamily a */ /** * @api {post} /test * @apiVersion v1 * @apiFamily b */

The second one does not override the first and is shown separately.

Example 2:
/** * @api {post} /v1/test * @apiVersion v1 * @apiFamily a */ /** * @api {post} /v2/test * @apiVersion v2 * @apiFamily a */

The second one is combined with the first and is shown under version.

@apiHeaderValue
Usage:
@apiHeaderValue [{type}] value [description]

Describes custom header value.

Example:
/** * @apiHeaderValue {String} value String value */

@apiNote
Usage:
@apiNote title

Adds note section that describes some additional information. Used instead of @api annotation with all its allowed annotations.

Example:
/** * @apiNote Note * @apiVersion v1 * @apiDescription Some description */

@apiParamPrefix
Usage:
@apiParamPrefix prefix

Prefixes all following @apiParams with prefix.

Allows to reuse lists of @apiParams between different doc blocks.

Example:
/** * @apiDefine sharedParams * @apiParam x * @apiParam y */ /** * @api {post} test1 * @apiParamPrefix body. * @apiUse sharedParams */ /** * @api {post} test2 * @apiParamPrefix payload. * @apiUse sharedParams */

@apiParamValue
Usage:
@apiParamValue [{type}] value [description]

Describes custom parameter value.

Example:
/** * @apiParamValue {String} value String value */

@apiSchema
Usage:
@apiSchema [(group)] {jsonschema=pathToFile[#internal.path]} @apiParam
@apiSchema [(group)] {swagger=pathToFile#internal.path.to.api} operationNickname
@apiSchema [(group)] {swagger=pathToFile#internal.path.to.model} @apiParam

Uses external schema to fill doc block.

jsonschema allows to use $ref definitions within schema. swagger generates doc block by api operation or @apiParam list by model.

Example:
/** * @apiSchema {jsonschema=./api.json#definitions.entity.get} @apiParam */

@apiSampleRequestVariable
Usage:
@apiSampleRequestVariable [(namespace)] [{responsePath}] field[=defaultValue]

Defines the "Send sample request" plug-in variable which can be used globally through placeholders in the values of @apiHeader and @apiParam.

jsonschema allows to use $ref definitions within schema. swagger generates doc block by api operation or @apiParam list by model.

Example:
/** * @apiSchema {jsonschema=./api.json#definitions.entity.get} @apiParam */
